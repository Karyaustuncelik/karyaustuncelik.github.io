Personal · Python / Networking
Drone Sensor Gateway System
2025

Multi-component IoT simulation built in pure Python: lightweight headless sensor nodes periodically send temperature and humidity readings to a drone gateway over TCP, retrying on connection failure.

The drone gateway acts as the edge processing unit — it maintains a 10-reading rolling window per sensor, calculates averages, detects anomalies, and forwards summaries to a central server. A simulated battery drains 1% per second; at 15% the drone pauses forwarding and queues data instead; at 100% it flushes the queue and resumes.

The central server features a live Tkinter GUI with real-time sensor tables, per-sensor graphs, and an event log. Zero data loss guaranteed by the queuing mechanism.

Academic · CS415
Deception Detection in Diplomacy Dialogues
Nov 2025 – Dec 2025

Worked on the Diplomacy Deception Dataset (17,289 messages across 12 games, 94.9% truthful / 5.1% deceptive). Framed as binary classification under strong class imbalance, prioritising deceptive-class F1 over accuracy.

Built a multi-stage feature pipeline combining TF–IDF (unigrams + bigrams), VADER sentiment scores (4 dimensions), and game-state metadata (year, season, score delta, speaker/receiver). Trained class-weighted logistic regression with validation-set threshold tuning — best tuned Lie F1 = 0.176.

Extended to fine-tuned RoBERTa (Lie F1 = 0.137, ROC–AUC = 0.656) and LightGBM on RoBERTa [CLS] embeddings. Conducted full ablation study across text-only, text+sentiment, and full-feature configurations.

Academic · CS303
2-Player Battleship on FPGA
2024

Implemented a complete 2-player Battleship game on an FPGA board using Verilog HDL.

Game logic is driven by a Finite State Machine (FSM) handling all phases: ship placement, turn management, coordinate input validation, hit/miss detection, ship sinking, and win condition evaluation. Player inputs are mapped to physical switches and buttons for coordinate selection and confirmation. LEDs provide immediate visual feedback for hits and misses. Academic · PROJ201
Nonlinear Perfect Code Construction
Jan 2024

Group research project supervised by Prof. Ferruh Özbudak, exploring error-correcting codes — specifically perfect codes — with no prerequisite background assumed.

Constructed a binary 1-error-correcting nonlinear perfect code of 2,048 codewords using the Hamming-based construction C2n+1 = {(v, v+c, p(v)+f(c)) : v ∈ 𝔽₂ⁿ, c ∈ Cₙ} with parameters r=3, n=7, k=4. The nonlinear function f was mapped arbitrarily to break linearity while preserving the perfect code property.

Implemented in Python with no external libraries; the full 2,048 codewords were generated and verified exhaustively, and the complete listing was included as an appendix.
